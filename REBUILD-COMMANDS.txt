============================================================
 OpenIDX - Memory-Safe Rebuild (Windows / PowerShell)
 Build affected services ONE AT A TIME to avoid OOM.
 Run every command from the repo root.
============================================================


------------------------------------------------------------
 AFFECTED SERVICES (this update)
------------------------------------------------------------
 REBUILD (backend code changed):
   identity-service
   oauth-service
   admin-api
   access-service
   governance-service
   provisioning-service
   audit-service

 REBUILD (frontend changed):
   admin-console

 DO NOT REBUILD:
   gateway-service   (no code change affects it)

 CONFIG-ONLY FIX (no rebuild, just recreate containers):
   DEFAULT_ORG_FALLBACK=true  -> fixes "organization empty"


------------------------------------------------------------
 STEP 0 - Get the latest code
------------------------------------------------------------
git pull


------------------------------------------------------------
 STEP 1 - Limit build parallelism (avoids OOM)
------------------------------------------------------------
$env:DOCKER_BUILDKIT = "1"
$env:BUILDKIT_MAX_PARALLELISM = "1"


------------------------------------------------------------
 STEP 2 - Build affected services, ONE AT A TIME
           (gateway-service is intentionally skipped)
------------------------------------------------------------
docker compose -f deployments/docker/docker-compose.yml build identity-service
docker compose -f deployments/docker/docker-compose.yml build oauth-service
docker compose -f deployments/docker/docker-compose.yml build admin-api
docker compose -f deployments/docker/docker-compose.yml build access-service
docker compose -f deployments/docker/docker-compose.yml build governance-service
docker compose -f deployments/docker/docker-compose.yml build provisioning-service
docker compose -f deployments/docker/docker-compose.yml build audit-service


------------------------------------------------------------
 STEP 3 - Build the frontend (separate, Node build)
------------------------------------------------------------
docker compose -f deployments/docker/docker-compose.yml build admin-console


------------------------------------------------------------
 IF A SINGLE BUILD STILL RUNS OUT OF MEMORY
   Cap memory for that one build, then prune between builds.
   (repeat the pattern for whichever service failed)
------------------------------------------------------------
docker compose -f deployments/docker/docker-compose.yml build --memory 4g --memory-swap 8g identity-service
docker builder prune -f


------------------------------------------------------------
 STEP 4 - Apply everything (new images + the compose env fix)
           --no-build so it does NOT rebuild again
------------------------------------------------------------
docker compose -f deployments/docker/docker-compose.yml up -d --no-build


------------------------------------------------------------
 STEP 5 - Verify the "organization empty" fix is live
------------------------------------------------------------
docker compose -f deployments/docker/docker-compose.yml exec identity-service printenv DEFAULT_ORG_FALLBACK
   (expected output:  true)

docker compose -f deployments/docker/docker-compose.yml ps


------------------------------------------------------------
 STEP 6 - (optional) Run the org doctor to confirm tenancy
------------------------------------------------------------
docker compose -f deployments/docker/docker-compose.yml exec postgres sh -lc "PGOPTIONS='-c row_security=off' psql -U openidx -d openidx -c 'select id,slug,status from organizations;'"


============================================================
 NOTES
------------------------------------------------------------
 - If you use docker-compose.prod.yml instead, replace
   "docker-compose.yml" with "docker-compose.prod.yml" in
   every command above.
 - For a REAL multi-tenant (subdomain-per-org) deploy, set
   DEFAULT_ORG_FALLBACK=false instead (env or compose).
 - Migrations/seed are handled by the migrate + seed services
   on "up"; you do NOT need to rebuild them for this update.
============================================================
